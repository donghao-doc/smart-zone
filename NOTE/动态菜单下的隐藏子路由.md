# 动态菜单下的隐藏子路由

## 目标
让详情页可被按钮跳转访问，但不出现在左侧菜单中，同时面包屑展示
`财务管理 / 合同管理 / 合同详情` 这类层级。

## 方案
1) 路由继续由菜单数据动态生成（权限跟随菜单）。
2) 菜单项新增 `menu?: boolean`：
   - `menu: false` 仅用于隐藏菜单展示。
   - 路由仍会被生成，因为菜单数据中仍包含该项。
3) 详情页作为父菜单的子节点挂载：
   - 例：`合同详情` 作为 `合同管理` 的子菜单。

## 不用静态路由的原因
静态路由会绕开菜单权限控制，导致无权限用户也能直接访问页面。

## 实现要点
- 菜单渲染时过滤 `menu === false`，仅隐藏展示。
- 路由生成仍遍历完整菜单树（包含隐藏项）。
- 面包屑基于菜单树回溯父链，隐藏子菜单仍可展示父级层级。

## 示例结构（菜单数据）
```ts
{
  label: '财务管理',
  key: '/finance',
  children: [
    {
      label: '合同管理',
      key: '/finance/contract',
      children: [
        {
          label: '合同详情',
          key: '/finance/surrender',
          menu: false,
        },
      ],
    },
  ],
}
```

## 检查清单
- [ ] 详情页挂在父菜单的子节点下
- [ ] `menu: false` 仅控制菜单展示，不影响路由生成
- [ ] 面包屑基于菜单树生成父级层级
